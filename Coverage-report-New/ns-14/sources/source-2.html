2  * This file contains an implementation of an integer only stack which is extremely quick and
3  * lightweight. In terms of performance it can outperform java.util.ArrayDeque (Java's fastest stack
4  * implementation) by a factor of 50! See the benchmark test below for proof. However, the downside
5  * is you need to know an upper bound on the number of elements that will be inside the stack at any
6  * given time for it to work correctly.

8  * @author William Fiset, william.alexandre.fiset@gmail.com

34   @Override
35   public Integer peek() {

40   @Override

43   }
44 
45   // Make sure you check that the stack is not empty before calling pop!
46   @Override
47   public Integer pop() {

49   }
